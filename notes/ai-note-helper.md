# AI智能笔记助手（纯后端API版）- 项目完整总结
## 一、项目核心信息
### 1. 项目定位
面向AI应用/后端方向的轻量化实战项目，核心是将阿里云OCR（文本识别）和大模型API（文本总结/问答）封装为标准化后端接口，解决“图片/PDF笔记→文本提取→AI智能处理”的实际需求，同时通过RAG技术杜绝AI幻觉问题。

### 2. 核心技术栈
| 技术/工具 | 作用 | 核心版本（适配CentOS7） |
|-----------|------|------------------------|
| FastAPI | 高性能后端API框架，封装业务接口 | ≥0.80.0 <0.90.0 |
| Uvicorn | ASGI服务器，运行FastAPI应用 | 0.16.0（兼容Python3.6） |
| Requests | 调用阿里云OCR/大模型HTTP API | ≥2.27.0 <2.28.0 |
| PyMySQL | 连接MySQL数据库，持久化笔记数据 | ≥1.0.0 <1.1.0 |
| Python-multipart | 处理文件上传（FastAPI依赖） | ≥0.0.5 <0.0.6 |
| MySQL | 存储笔记ID、文件名、OCR文本、AI总结 | 适配CentOS7的MariaDB/MySQL均可 |

### 3. 核心解决的问题
1. 图片/PDF笔记的文本提取（基于阿里云OCR）；
2. AI总结笔记内容（基于通义千问qwen-turbo）；
3. 基于笔记原文的精准问答（RAG技术，杜绝AI幻觉）；
4. 提供标准化API接口，支持前端/客户端调用。

## 二、项目核心功能（接口清单）
| 接口路径 | 请求方法 | 功能描述 | 入参/出参说明 |
|----------|----------|----------|---------------|
| `/api/health` | GET | 服务健康检查 | 出参：`{"code":200,"msg":"服务正常运行","data":{"time":"时间戳"}}` |
| `/api/ocr` | POST | 上传文件（图片/PDF），OCR识别文本 | 入参：上传文件（FormData格式）；出参：`{"code":200,"msg":"识别成功","data":{"note_id":"唯一ID","ocr_text":"识别文本"}}` |
| `/api/summary` | POST | 根据笔记ID生成AI总结 | 入参：`{"note_id":"笔记唯一ID"}`；出参：`{"code":200,"msg":"总结成功","data":{"note_id":"ID","summary":"分点总结文本"}}` |
| `/api/chat` | POST | 基于笔记原文的RAG问答 | 入参：`{"note_id":"ID","question":"用户问题"}`；出参：`{"code":200,"msg":"问答成功","data":{"answer":"仅基于原文的回答"}}` |
| `/docs` | GET | FastAPI自动生成的Swagger文档 | 可视化测试所有接口，无需写前端代码 |

## 三、项目实现步骤（关键流程）
### 1. 环境准备（适配CentOS7）
1. 安装依赖：修改`requirements.txt`适配清华源，降低版本兼容Python3.6，执行`pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`；
2. 数据库准备：安装MariaDB/MySQL，创建`ai_notes`数据库（`CREATE DATABASE ai_notes DEFAULT CHARSET utf8mb4;`）；
3. 阿里云配置：获取API密钥（需开通OCR和大模型调用权限）。

### 2. 核心代码逻辑
1. 初始化FastAPI：配置跨域（CORS），解决前端调用问题；
2. 数据库层：初始化`notes`表，实现笔记数据的增/查操作；
3. 阿里云API调用：封装OCR识别（图片转16进制）、大模型调用（构造Prompt）函数；
4. 接口层：封装4个核心接口，处理异常（如笔记不存在、API调用失败）；
5. 服务启动：通过Uvicorn启动服务，监听0.0.0.0:8000（支持公网访问）。

### 3. 部署与访问
1. 启动服务：`nohup python3 main.py > ai-note-api.log 2>&1 &`（后台运行）；
2. 进程管理：通过`lsof -i:8000`查进程PID，`kill -9 PID`终止进程；
3. 公网访问：阿里云控制台开放8000端口（安全组规则填`8000/8000`），访问`http://公网IP:8000/docs`测试接口。

## 四、项目核心亮点（简历重点）
1. AI能力落地：将阿里云OCR和大模型API封装为标准化后端接口，实现“识别-总结-问答”全流程；
2. 解决AI幻觉：采用RAG技术，限定大模型仅基于笔记原文生成内容，保证回答准确性；
3. 工程化基础：接口统一返回格式（code/msg/data）、处理跨域、提供自动接口文档，符合企业开发规范；
4. 环境适配：针对CentOS7/Python3.6适配依赖版本，解决国内源同步、服务启动等实际部署问题。

## 五、常见问题与解决方案
| 问题类型 | 报错现象 | 解决方案 |
|----------|----------|----------|
| 依赖版本不兼容 | 安装时提示“找不到对应版本” | 降低依赖版本（如uvicorn→0.16.0、requests→2.27.0），适配清华源； |
| MySQL连接失败 | “Can't connect to local MySQL server” | 启动MariaDB服务（`systemctl start mariadb`），检查socket路径； |
| Python版本兼容 | “asyncio.run() is not defined” | 降级uvicorn到0.16.0（兼容Python3.6），或升级Python到3.8+； |
| 公网无法访问 | 浏览器访问接口超时 | 阿里云安全组开放8000端口，格式填`8000/8000`，授权对象`0.0.0.0/0`； |
| 进程管理 | 服务启动后无法终止 | 通过`ps -ef | grep main.py`查PID，执行`kill -9 PID`强制终止； |

## 六、新手简化落地建议
1. 优先本地验证：放弃服务器部署，在Windows/macOS（Python3.8+）上跑通核心AI逻辑，跳过数据库/依赖适配；
2. 最小化接口：只保留`/ocr_summary`合并接口（上传文件→识别+总结），减少代码复杂度；
3. 简历聚焦核心：不提服务器配置、依赖排错等细节，重点写“AI接口封装、RAG解决幻觉、提供接口文档”。

## 七、技术边界说明（AI后端视角）
1. 无需精通前端：可通过FastAPI自动生成的`/docs`页面完成接口演示，无需写HTML/CSS/JS；
2. 核心能力重点：深耕AI API调用、后端接口设计、数据持久化，服务器配置仅需了解基础；
3. 协作关键点：明确接口入参/出参格式、解决跨域问题，能对接前端调用即可。
